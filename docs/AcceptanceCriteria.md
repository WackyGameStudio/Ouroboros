## 2. 공통 Definition of Done

기능은 다음 조건을 모두 만족해야 완료다.

- 책임 소유 클래스가 하나이고 성공·거부·오류 결과를 구분한다.
- 무효 요청은 상태와 풀 사용량에 부작용을 만들지 않는다.
- ScriptableObject 원본을 변경하지 않고 상태 확정 후 이벤트를 1회만 발행한다.
- 선택·사망·재시작 생명주기와 풀 반환 초기화를 처리한다.
- 최소 EditMode 또는 PlayMode 테스트가 있으며 예상하지 않은 Console Error/Exception이 없다.
- 반복 실행 경로의 GC Alloc을 확인하고 아트 없이도 규칙을 검증할 디버그 표현이 있다.
- 아트 적용 후 텔레그래프·판정·상태 표현이 실제 규칙과 일치한다.
- 데이터 기능은 ID 고유성, 범위·유한 수, 잘못된 설정의 시작 차단, 데이터 버전 기록을 검증한다.
- UI는 마우스·키보드와 `timeScale=0`에서 동작하고 규칙 객체를 직접 수정하지 않는다.

## 4. 자동 테스트 필수 행렬

### EditMode

- 몸통 공통 화력: L=0,4,5,9,10,20,40,64
- 폭발: N=3 거부, 4→2, 5→2, 10→3, 64→20
- 체인: 생성, 중간/꼬리/전체 절단, 예약 소비와 이중 제거 방지
- 선택 큐: Body 2 + LevelUp 2 → Body, Body, Level, Level
- 조각·경험치: 경계값, 다중 요청, 초과분 보존, 64 보류 재개
- 업그레이드: 후보 중복 없음, 최대 단계 제외, 초반 계열 보정
- 결과 코드: 무효 요청 전후 상태 해시 동일
- 데이터: null, 중복 ID, 0 요구량, 음수, NaN, Infinity 거부

### PlayMode

- 입력 Map 전환과 복귀 입력 미재생
- 선택 중 적·탄·픽업·타이머 정지
- 자동 공격 최근접·동률·무대상·정예 우선
- 역할별 중첩·제거·예고·피해 0 규칙
- 머리+몸통 동시 피격, 절단+폭발, 사망+픽업/폭발
- 재시작 후 SO 원본, 이벤트, 풀, `Time.timeScale` 초기화

## 5. 빌드 게이트

| 게이트 | 합격 기준 |
| --- | --- |
| G0 — 3분 코어 세로 슬라이스 | 시작 역할 2개→이동/자동 공격→몸통 성장→절단→폭발→화력 감소→재성장→사망/재시작. 진행 막힘·규칙 오류·Exception 0, 재시작 초기화 |
| G1 — 7분 빌드 | 적 4종과 정예 2회 등장, 역할·업그레이드·폭발의 동시 요청 규칙 준수, 평균 적 수와 위험 신호 판독성 기록 |
| G2 — 10분 빌드 | 시작 선택부터 보스 처치/사망 결과까지 완주, 재시작 후 재진입, Console Error/Exception 0, 동시 사건 테스트 통과 |

## 6. 수동 플레이테스트 합격 기준

- 테스터 70% 이상이 몸통 유지와 소비 사이에서 실제 고민했다고 응답한다.
- 폭발 입력 전에 소비량과 범위를 예측할 수 있다.
- 머리 피해와 몸통 절단 원인을 구분한다.
- 폭발 후 재성장이 단순 무력감이 아니라 다음 반격 준비로 느껴진다.
- 기록에는 Build ID, 플랫폼, Data Version, Run Seed, 결과 시각, 첫 폭발, 최대/최종 몸통, 절단/폭발 손실, 가독성 실패 시각을 포함한다.
